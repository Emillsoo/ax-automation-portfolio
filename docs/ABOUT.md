## 약 500자

저는 의료기관의 업무와 시스템 환경을 함께 이해하는 AX 엔지니어입니다. 의료기관 IT 조직에서 인프라·운영 업무를 경험한 뒤 n8n, Python, Robot Framework, Java와 SQL을 활용한 업무자동화와 EMR 시스템 연동으로 역할을 확장했습니다. 지출결의서 RPA, 재원환자 자격 불일치 알림, 의료문서 AI 초안, EMR 내부 API 등을 직접 구현했으며, 개발한 기능을 서버에 배포하고 모니터링하며 장애 원인을 분석해 재발 방지 구조로 개선했습니다. 폐쇄망 패키지 설치와 방화벽·DNS 문제를 해결해 개발 PC의 배치를 운영 서버로 이관한 경험도 있습니다. 자동화에서는 API의 성공 응답보다 실제 업무 결과를 중요하게 봅니다. 이중 필터, 0건 조기 종료, 결과 재조회, 실패 알림과 인적 검토를 설계해 의료·금전 업무의 위험을 낮춥니다. 장애를 개인 경험으로 끝내지 않고 체크리스트와 배포 원칙으로 남깁니다. 현업 문제를 기술로 바꾸고 안정적인 운영까지 책임지는 엔지니어가 되겠습니다.

## 약 1,000자

저는 의료기관의 현업 업무, 레거시 EMR, 서버·네트워크 환경을 함께 이해하고 실제 업무 개선으로 연결하는 AX 엔지니어입니다. 2023년부터 의료기관 IT·디지털혁신 조직에서 근무하며 인프라와 운영을 경험했고, 이후 n8n, Python, Robot Framework, Java, SQL을 활용한 업무자동화와 시스템 연동으로 역할을 확장했습니다.

업무를 자동화할 때는 화면을 그대로 따라 만드는 것보다 화면이 적용하는 비즈니스 규칙과 예외 조건을 먼저 분석합니다. 재원환자 자격 불일치 자동화에서는 EMR 화면의 판단 기준을 SQL로 재현하고 최신 결과, 기관 범위, 미수신 데이터와 시간대 문제를 방어했습니다. 지출결의서 자동화에서는 외부 기반의 최소 골격 위에 Flask API, 작업 상태 관리, Robot Framework 화면 제어, 실시간 진행 전달과 예외 처리를 구현했습니다. 의료문서 AI 초안 자동화에서는 긴 원문을 항목별로 처리하고 EMR 임시저장과 의료진 검토 단계를 두어 AI가 최종 판단을 대신하지 않도록 설계했습니다. 또한 Java·iBatis 기반 EMR 컴포넌트를 직접 개발해 화면 안에 있던 기능을 내부 자동화와 타 시스템에서 재사용할 수 있는 API로 확장했습니다.

저의 강점은 개발과 운영을 분리해서 보지 않는다는 점입니다. 폐쇄망의 패키지 설치, 32비트 COM, 방화벽·DNS·프록시 문제를 해결해 배치를 운영 서버로 이관했고, 자동화 장애에서는 성공 응답만 믿지 않고 실제 데이터를 재조회하는 검증과 모니터링을 도입했습니다. 이중 필터, 0건 조기 종료, 전체 중단, 실패 알림과 롤백을 공통 원칙으로 만들고 형상관리와 배포 절차도 문서화했습니다.

앞으로도 의료기관의 업무를 정확히 이해하고, n8n·스크립트·API·AI를 적절히 조합해 현업이 신뢰할 수 있는 자동화를 만들고 싶습니다. 빠르게 기능을 만드는 것뿐 아니라 개인정보 보호, 실패 가능성, 운영 가시성과 재발 방지까지 책임지는 AX 엔지니어로 기여하겠습니다.
